# SelfOdds 项目更新记录

本文件记录面向用户的重要能力、行为变化与可靠性修复。日期使用北京时间。

## 2026-07-21 · DeepSeek 稳定性与模式化 Schema

### 修复

- 修复 DeepSeek 在项目模式返回 `success_probability: null` 时整份结果被拒绝的问题。服务端会使用可建设性、证据分和外部视角先验进行有限补全。
- 修复任务执行和 Agent 审计模式把 `evidence_ledger` 返回为 `null` 或包含非法条目时频繁进入本地降级的问题。
- 修复模型返回的评估类型可能与用户选择模式不一致的问题；最终类型现在由服务端请求模式确定。
- 将 DeepSeek 最大输出从 3200 提升到 5000 tokens，避免复杂需求分析与证据账本被截断为无效 JSON。
- 将模型请求超时从 45 秒提升到 75 秒。

### 可靠性

- 增加模式感知的输出标准化：项目模式保留合法仓库证据；任务与 Agent 模式只保留合法的 `USER_INPUT` 证据。
- 无效证据条目会被丢弃，不会被改写成“已观察事实”。核心概率、风险、验证步骤和项目评分仍通过严格 Schema。
- API 错误现在区分：未配置、鉴权失败、限流、超时、上游异常和输出协议错误。
- 新增回归测试，覆盖 `success_probability: null`、`evidence_ledger: null`、任务模式和 Agent 审计模式。
- 真实 DeepSeek 验证结果：任务执行与 Agent 审计均返回 HTTP 200 和 `source: agent`。

## 2026-07-21 · 公开评判标准与自动校准

### 新增

- 项目机会评分采用固定六维权重：需求 25%、势能 15%、差异化 20%、可建设性 20%、分发 10%、证据 10%。
- 增加 0/25/50/75/100 公开证据锚点、A–D 等级与确定性总分计算。
- 增加精准需求分析：目标用户、核心问题、当前替代方案、紧迫度、正反证据、未知信息与可证伪假设。
- 增加 `OBSERVED`、`INFERRED`、`UNKNOWN` 证据账本。
- 增加 7 天 GitHub 趋势预测、预测区间、自动结算合约、Brier Score 和校准分。
- 接入 D1 预测记录与自动结算接口。

## 2026-07-21 · GitHub 研究与三种评估模式

### 新增

- 自动读取 GitHub 仓库元数据、README、根目录、语言、许可证和最近活跃时间。
- 增加项目机会、任务执行、Agent 审计三种模式。
- Agent 审计可检查用户编写的提示词、计划和可见输出，并生成推理缺口、对抗测试、验证步骤与改进指令。
- 项目机会分与最小实验执行成功率分离，避免把 Star 热度直接当成产品价值。

## 2026-07-21 · Preflight Agent MVP

### 新增

- 中英文 SelfOdds 产品界面。
- DeepSeek 与 OpenAI 服务端自动路由。
- `SENSE → CHALLENGE → DECIDE → GUARD` 四阶段决策闭环。
- `AUTORUN`、`REVIEW`、`ESCALATE` 三档自治路由与服务端守门器。
- AI 不可用时使用明确标记的本地规则，不把降级结果伪装成模型结果。
